# 코드 리뷰 가이드

이 스킬은 review-report 플러그인의 코드 리뷰 모범 사례와 4개 전문 에이전트 활용 방법을 안내합니다.

## 에이전트 구성

| 에이전트 | 대상 | 역할 |
|---------|-----|------|
| screen-component-reviewer | QA/기획 | 유저 관점 페이지 분류 및 컴포넌트/기능 변경사항 분석 |
| side-effect-reviewer | 개발/QA | 수정된 코드의 Side Effect 분석 및 주의사항 안내 |
| code-quality-reviewer | 개발자 | 코드 품질, 패턴, 개선점 피드백 |
| qa-checklist-reviewer | QA | 이번 작업에 대한 테스트 체크리스트 생성 |

## 4개 전문 에이전트 소개

### screen-component-reviewer (파란색)
**대상**: QA, 기획자
**역할**: 유저 관점에서 페이지를 분류하고 각 페이지 내 변경된 컴포넌트와 기능을 분석

**활용 시점**:
- UI 컴포넌트 변경 시
- 화면 레이아웃 수정 시
- 기능 로직 변경 시

**주요 검토 항목**:
- 유저가 보는 페이지 단위 분류
- 페이지별 변경 컴포넌트 목록
- 변경 전/후 비교

---

### side-effect-reviewer (주황색)
**대상**: 개발자, QA
**역할**: 코드 변경으로 인해 발생할 수 있는 Side Effect 분석

**활용 시점**:
- 전역 상태 수정 시
- API 호출 패턴 변경 시
- 이벤트 핸들러 수정 시

**주요 검토 항목**:
- 주의 필요 Side Effect
- 영향받는 코드/기능
- 확인 체크리스트

---

### code-quality-reviewer (녹색)
**대상**: 개발자
**역할**: 코드 품질, 패턴, 잠재적 이슈 분석 및 개선점 제안

**활용 시점**:
- 모든 코드 변경 시
- 리팩토링 작업 시
- 새 기능 추가 시

**주요 검토 항목**:
- 개선 필요/권장 항목
- 코드 예시
- 품질 지표

---

### qa-checklist-reviewer (청록색)
**대상**: QA
**역할**: 이번 작업에서 QA가 확인해야 할 테스트 항목 생성

**활용 시점**:
- 기능 개발 완료 후
- 버그 수정 완료 후
- 릴리스 전 테스트 준비 시

**주요 검토 항목**:
- 기능/입력값/UI/시나리오별 테스트 체크리스트
- 테스트 방법 및 예상 결과
- 특별 주의 사항

---

## 사용 방법

### 전체 리뷰 실행
```
/review-report
```
4개 에이전트가 병렬로 실행되어 종합 리포트를 생성합니다.

### 특정 경로 리뷰
```
/review-report --target src/components/Button
```

### 특정 타입만 실행
```
/review-report --type screen      # 페이지별 변경사항
/review-report --type side-effect # Side Effect 분석
/review-report --type quality     # 코드 품질 (개발자용)
/review-report --type qa          # QA 체크리스트
```

## 출력 형식

리뷰 완료 시 다음 항목을 포함한 종합 리포트가 생성됩니다:

- 페이지별 변경사항
- Side Effect 주의사항
- 코드 품질 피드백 (개발자용)
- QA 테스트 체크리스트

## 모범 사례

1. **점진적 리뷰**: 큰 변경은 여러 작은 PR로 분리
2. **자동화 활용**: CI/CD에 lint, test 통합
3. **문서화**: 복잡한 로직은 주석 또는 문서 추가
4. **테스트 우선**: 변경사항에 대한 테스트 작성
5. **협업**: 리뷰 결과를 바탕으로 팀 토론
