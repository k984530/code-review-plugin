# 코드 리뷰 가이드

이 스킬은 review-report 플러그인의 코드 리뷰 모범 사례와 6개 전문 에이전트 활용 방법을 안내합니다.

## 코드 리뷰 원칙

### 1. 건설적인 피드백
- 문제점만 지적하지 않고 개선 방안 제시
- 긍정적인 부분도 언급
- 명확하고 구체적인 피드백 제공

### 2. 일관된 기준 적용
- 프로젝트 코딩 컨벤션 준수 확인
- 팀 합의된 패턴 적용 검증
- 개인 취향이 아닌 객관적 기준 사용

### 3. 사용자 관점 고려
- 기술적 완성도와 UX 모두 고려
- 실제 사용 시나리오 기반 검토
- 접근성 및 성능 영향 평가

## 6개 전문 에이전트 소개

### screen-component-reviewer (파란색)
**역할**: 화면/컴포넌트/기능 단위 분석

**활용 시점**:
- UI 컴포넌트 변경 시
- 화면 레이아웃 수정 시
- 기능 로직 변경 시

**주요 검토 항목**:
- 화면 레이아웃 영향
- Props/State 변경 호환성
- 기능 로직 정확성

---

### side-effect-reviewer (주황색)
**역할**: 부작용 및 의도치 않은 영향 분석

**활용 시점**:
- 전역 상태 수정 시
- API 호출 패턴 변경 시
- 이벤트 핸들러 수정 시

**주요 검토 항목**:
- 전역 상태 변경 영향
- 메모리 누수 가능성
- 비동기 작업 정리

---

### code-quality-reviewer (녹색)
**역할**: 코드 품질, 가독성, 유지보수성 평가

**활용 시점**:
- 모든 코드 변경 시
- 리팩토링 작업 시
- 새 기능 추가 시

**주요 검토 항목**:
- 코딩 컨벤션 준수
- 중복 코드 제거
- 복잡도 관리
- 타입 안정성

---

### impact-analyzer (보라색)
**역할**: 변경 영향 범위 분석

**활용 시점**:
- 공유 컴포넌트 수정 시
- 유틸리티 함수 변경 시
- API 인터페이스 수정 시

**주요 검토 항목**:
- 의존성 그래프
- 영향받는 파일 목록
- 하위 호환성

---

### qa-checklist-reviewer (청록색)
**역할**: QA 체크리스트 기반 검증

**활용 시점**:
- 릴리스 전 최종 검토
- 테스트 커버리지 확인
- 품질 기준 충족 확인

**주요 검토 항목**:
- 테스트 커버리지
- 에러 핸들링
- 접근성
- 브라우저 호환성
- 성능 영향

---

### user-flow-reviewer (노란색)
**역할**: UX 관점 분석

**활용 시점**:
- 사용자 인터랙션 변경 시
- UI 피드백 수정 시
- 사용자 여정 영향 시

**주요 검토 항목**:
- 사용자 시나리오 영향
- UI/UX 일관성
- 상태 처리 (로딩/에러/빈 상태)

## 사용 방법

### 전체 리뷰 실행
```
/review-report
```
6개 에이전트가 병렬로 실행되어 종합 리포트를 생성합니다.

### 특정 경로 리뷰
```
/review-report --target src/components/Button
```

### 특정 타입만 실행
```
/review-report --type quality
/review-report --type side-effect
/review-report --type impact
```

## 리뷰 결과 활용

1. **필수 수정 사항** 우선 처리
2. **권장 개선 사항** 검토 및 적용 결정
3. 팀 내 리뷰 결과 공유 및 논의
4. 반복되는 이슈는 코딩 가이드라인에 추가

## 모범 사례

1. **점진적 리뷰**: 큰 변경은 여러 작은 PR로 분리
2. **자동화 활용**: CI/CD에 lint, test 통합
3. **문서화**: 복잡한 로직은 주석 또는 문서 추가
4. **테스트 우선**: 변경사항에 대한 테스트 작성
5. **협업**: 리뷰 결과를 바탕으로 팀 토론
