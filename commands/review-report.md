# QA 테스트 가이드 생성

6개의 전문 분석 에이전트를 통해 코드 변경사항을 분석하고, QA가 무엇을 테스트해야 하는지 종합 리포트를 생성합니다.

## 리뷰 대상

{{#if target}}
대상 경로: `{{target}}`
{{else}}
대상: 최근 변경된 파일들 (git diff 또는 현재 디렉토리)
{{/if}}

## 실행 지침

{{#if type}}
### 단일 에이전트 실행
`{{type}}` 타입의 분석만 실행합니다.

해당 에이전트를 실행하여 분석 결과를 제공하세요:
{{#eq type "screen"}}
- `screen-component-reviewer` 에이전트 실행: 변경된 화면/기능 분석
{{/eq}}
{{#eq type "side-effect"}}
- `side-effect-reviewer` 에이전트 실행: 부작용 가능성 분석
{{/eq}}
{{#eq type "quality"}}
- `code-quality-reviewer` 에이전트 실행: 잠재적 버그 분석
{{/eq}}
{{#eq type "impact"}}
- `impact-analyzer` 에이전트 실행: 테스트 범위 분석
{{/eq}}
{{#eq type "qa"}}
- `qa-checklist-reviewer` 에이전트 실행: 테스트 체크리스트 생성
{{/eq}}
{{#eq type "user-flow"}}
- `user-flow-reviewer` 에이전트 실행: 사용자 시나리오 분석
{{/eq}}
{{else}}
### 전체 분석 실행

다음 6개의 에이전트를 **반드시 병렬로** 실행하세요.
**중요**: 단일 메시지에서 6개의 Task 도구 호출을 동시에 수행해야 합니다.

1. **screen-component-reviewer**: 변경된 화면/기능 식별
2. **side-effect-reviewer**: 연관 기능 부작용 분석
3. **code-quality-reviewer**: 잠재적 버그 포인트 분석
4. **impact-analyzer**: 테스트 범위 및 우선순위 분석
5. **qa-checklist-reviewer**: 테스트 체크리스트 생성
6. **user-flow-reviewer**: 사용자 시나리오 기반 테스트 케이스 도출

각 에이전트에게 전달할 프롬프트:
```
{{#if target}}
리뷰 대상 경로: {{target}}
{{else}}
git diff를 사용하여 최근 변경사항을 분석하거나, 현재 디렉토리의 주요 파일들을 리뷰하세요.
{{/if}}

코드 변경사항을 분석하고 QA가 테스트해야 할 항목을 도출하세요.
```
{{/if}}

모든 에이전트의 분석이 완료되면, 다음 형식으로 **QA용 종합 테스트 가이드**를 작성하세요:

- **개발 변경 요약**: [개발자가 무엇을 변경했는지 1-2문장으로 설명]
